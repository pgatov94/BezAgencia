# БезАгенция — самостоятелен сайт (извън Claude)

Този проект е същият сайт, който изгради дизайна дотук, но вече е истинско,
самостоятелно уеб приложение с истинска база данни (Supabase). Работи за
всеки посетител — без нужда от Claude акаунт.

Има три стъпки: (1) база данни, (2) сложи ключовете в кода, (3) качи сайта.
Не е нужно да можеш да програмираш — просто следвай стъпките.

---

## Стъпка 1 — Създай безплатна база данни в Supabase

1. Отиди на **https://supabase.com** → „Start your project" → регистрирай се
   (може direct с Google/GitHub акаунт).
2. Създай нов проект (New project): избери си име (напр. `bezagencia`) и
   парола за базата (запази я някъде, няма да ти трябва често).
3. Изчакай ~1-2 минути, докато Supabase подготви проекта.
4. Отвори секцията **SQL Editor** (в лявото меню) → New query.
5. Отвори файла `sql/schema.sql` от тази папка, копирай **целия** му
   съдържание, постави го в SQL Editor и натисни **Run**.
   Това създава 6-те таблици, от които сайтът има нужда (оферти, плащания,
   запитвания, отзиви, ваучери, абонати за бюлетина).
6. Отиди на **Project Settings → API** (иконка зъбно колело долу вляво →
   API). Ще видиш две неща, които ти трябват:
   - **Project URL** (изглежда като `https://abcdefgh.supabase.co`)
   - **anon public key** (дълъг низ от букви/цифри)

---

## Стъпка 2 — Сложи ключовете в кода

Отвори файла `src/supabaseClient.js` и замени:

```js
const SUPABASE_URL = "https://ТВОЯ-ПРОЕКТ.supabase.co";
const SUPABASE_ANON_KEY = "ТВОЯ-ANON-PUBLIC-KEY";
```

с истинските стойности от Стъпка 1 (точка 6). Запази файла.

Ако искаш, можеш и да смениш паролата на админ панела — тя е в
`src/App.jsx`, търси реда:

```js
const ADMIN_PASSCODE = "smeni-me-2026";
```

## Стъпка 3 — Качи сайта на живо (Vercel, безплатно)

Най-лесният начин, без терминал:

1. Качи цялата тази папка в нов **GitHub** repository (ако нямаш GitHub
   акаунт, регистрирай се безплатно на github.com — само провлачваш
   файловете при "Add file → Upload files").
2. Отиди на **https://vercel.com** → регистрирай се (може direct с GitHub
   акаунта) → **Add New → Project** → избери repository-то, което качи.
3. Vercel автоматично разпознава, че е Vite проект — просто натисни
   **Deploy**.
4. След ~1 минута ще получиш истински публичен линк
   (нещо като `bezagencia.vercel.app`), който работи за всеки, навсякъде,
   без Claude акаунт.

### Ако предпочиташ през терминал вместо GitHub качване:

```bash
npm install
npm run dev        # за да тестваш локално на http://localhost:5173
npm run build       # прави финалната версия в папка dist/
```

После инсталирай Vercel CLI (`npm i -g vercel`) и изпълни `vercel` в тази
папка — ще те преведе през останалото стъпка по стъпка.

## Стъпка 4 — Истински имейли с лого (Resend)

Формсприй (стария начин) имаше вграден спам филтър (Formshield), който
понякога блокираше автоматичните ни известия. Затова сега имейлите минават
през собствена функция в кода (`api/send-email.js`) през **Resend** —
безплатно до 3000 имейла/месец, с пълен контрол над съдържанието
(включително логото).

1. Отиди на **https://resend.com** → регистрация (безплатно)
2. **Domains** → **Add Domain** → въведи `bezagencia.com`
3. Resend ще ти покаже DNS записи (обикновено 2-3 — SPF, DKIM, понякога DMARC)
   — добави ги при регистратора на домейна ти (същото място, където добави
   записите за Vercel). Изчакай верификацията (обикновено бързо, до час).
   **Докато домейнът не е верифициран, можеш да пращаш само до собствения
   си имейл** (този, с който си се регистрирал в Resend) — за истински
   клиенти е задължителна тази стъпка.
4. **API Keys** → **Create API Key** → копирай ключа (започва с `re_...`)
5. Във **Vercel** → твоя проект → **Settings → Environment Variables** →
   добави:
   - Name: `RESEND_API_KEY`
   - Value: (ключът от точка 4)
   - Environment: Production (и Preview, ако искаш)
6. **Redeploy** проекта (Vercel → Deployments → ⋯ → Redeploy), за да влезе
   в сила новата променлива.

Логото се взима автоматично от `public/logo.png` (вече е в проекта) —
показва се във всеки изходящ имейл (запитване, напомняне за плащане,
бюлетин, отзив).

## Стъпка 5 — Истинско плащане с карта (Stripe)

1. Отиди на **https://stripe.com** → регистрация
2. В началото си в **Test mode** (превключвател горе вдясно) — така може
   да тестваш без реални пари, преди да минеш на Live mode
3. **Developers → API keys** → копирай **Secret key** (започва с `sk_test_...`
   в тестов режим, или `sk_live_...` на живо)
4. Във Vercel → Environment Variables → добави:
   - Name: `STRIPE_SECRET_KEY`
   - Value: (secret key от точка 3)
5. За да се маркират плащанията автоматично като "платено" (без админът
   да го прави ръчно): **Developers → Webhooks → Add endpoint**
   - Endpoint URL: `https://bezagencia.com/api/stripe-webhook`
   - Events to send: избери `checkout.session.completed`
   - След създаване, копирай **Signing secret** (започва с `whsec_...`)
   - Добави го във Vercel като `STRIPE_WEBHOOK_SECRET`
6. Redeploy проекта

Когато си готов за истински пари (не тест), превключи Stripe на **Live
mode** и повтори точки 3 и 5 с истинските (live) ключове.

## Какво е различно спрямо версията в Claude

- Всички данни (оферти, плащания, запитвания, отзиви, ваучери, абонати)
  вече се пазят в Supabase, а не в паметта на Claude артефакта — затова
  вече работят за реални клиенти, не само за теб.
- „Запомни ме" в админ панела вече използва истинска памет на браузъра
  (localStorage), тъй като вече не сме в защитената среда на Claude.
- Имейлите минават през Resend (собствена функция в кода), не през
  Formspree — пълен контрол над съдържанието, включително логото.
- Плащанията с карта минават през истински Stripe Checkout, с автоматично
  маркиране като платено чрез webhook.
- Дизайнът, текстовете, wizard-ът за запитвания, логото — всичко е същото,
  без промяна.

## Ако искаш повече сигурност по-нататък

В момента админ панелът е защитен само с обикновена парола, вградена в
кода (символична защита). Ако искаш истинска сигурност (истински логин,
права за достъп на ниво база данни), кажи ми — мога да настроя истински
Supabase Auth логин и Row Level Security policies, описани накратко в
края на `sql/schema.sql`.
